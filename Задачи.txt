                    Задача: Тестирование формы регистрации на веб-приложении

Описание: На веб-приложении требуется зарегистрироваться, предоставив информацию о имени, электронной почте, пароле и дате рождения. Для обеспечения эффективного тестирования необходимо применить технику эквивалентного разбиения.

Требования к данным:
Имя: от 3 до 50 символов, только буквы и пробелы
Электронная почта: допустимый формат адреса электронной почты
Пароль: от 6 до 20 символов, должен содержать хотя бы одну букву в верхнем регистре, одну букву в нижнем регистре и одну цифру
Дата рождения: допустимый формат даты, возраст от 18 до 99 лет

Задача:
Разработайте набор тестовых случаев, применяя технику эквивалентного разбиения, для проверки корректной регистрации на веб-приложении.
Укажите различные классы эквивалентности для каждого поля ввода (имя, электронная почта, пароль, дата рождения) и опишите, какие тестовые случаи они охватывают.
Для каждого класса эквивалентности определите хотя бы по одному тестовому случаю, который бы проверял корректное поведение поля (например, успешную регистрацию) и один тестовый случай, который бы проверял некорректное поведение (например, неверный формат даты).
Предложите спецификации и ожидаемые результаты для каждого тестового случая.
Важно учесть различные комбинации данных и обеспечить покрытие основных сценариев регистрации с использованием техники эквивалентного разбиения.

                        2. Проверка электронной почты 

*согласно нескольким RFC и гуглу адрес электронной почты может иметь следующий вид:
Адрес электронной почты состоит из двух частей: локальной части (до символа @) и доменной части (после символа @).
Локальная часть может состоять из следующих символов:
- заглавные и строчные буквы английского алфавита (A-Z, a-z)
- цифры (0-9)
- специальные символы (только эти): ! # $ % & ' * + - / = ? ^ _ { | } ~`
- не может начинаться с точки "." и заканчиваться точкой и она не может стоять подряд
- кавычки можно использовать для включения в локальную часть недопустимых символов и пробелов
- локальная часть не должна быть более 64 символов
Доменная часть:
- заглавные и строчные буквы английского алфавита (A-Z, a-z)
- цифры (0-9)
- спецсимволы использовать нельзя
- дефис "-" можно использовать, но не в начале и не в конце
- точки "." можно использовать для разделения меток (example@test.privet.com)
- доменная часть допускается общей длиной до 255 символов (общая длина всего адреса не более 320 включая @)
- допустимо 2-63 букв после точки в названии каждой метки (часть домена отделенная точкой), например: .ru или .qwertyuiopasdfghjkl

Позитивные для локальной части:
    - test@test.ru
    - 1234@test.ru
    - ####@test.ru
Негативные для локальной части:
    - .test@test.ru - точка в начале локальной части

Позитивные для доменной части:
    - test@test.ru (повторяется, можно не использовать)
    - test@1234.ru
Негативные для доменной части:
    - test@####.ru - в доменной части запрещенные спец символы

Можно принять за класс эквивалентности разрешенную длину всего адреса электронной почты, а также отдельно локальной и доменной частей
Позитивные для всего адреса:
    - test@test.ru (повторяется, можно не использовать)
Негативные для всего адреса:
    - ввести адрес общей длиной более 320 символов

Такие же тесты повторить для локальной (2-64) и доменной частей (255)

Проверка даты

    2. Задача: Тестирование калькулятора для расчета площади прямоугольника

Описание: Необходимо протестировать калькулятор, который позволяет вычислить площадь прямоугольника по его длине и ширине. Для тестирования используйте технику граничных значений.

Требования к входным данным:
Длина прямоугольника: положительное число, от 1 до 100
Ширина прямоугольника: положительное число, от 1 до 100

Задача:
Разработайте несколько тестовых случаев с использованием техники граничных значений для проверки корректности расчета площади прямоугольника.
Для каждого параметра (длина и ширина) определите граничные значения и выберите значения, близкие к границам.
Сформулируйте тестовые случаи, которые проверяют:
 Случай, когда длина и ширина прямоугольника равны минимальным значениям.
 Случай, когда длина и ширина прямоугольника равны максимальным значениям.
 Случай, когда один из параметров находится на границе минимального значения, а другой - на границе максимального значения.
 Случай с произвольными значениями длины и ширины.
 Случаи с отрицательными параметрами

Граничные значения для длины 1 и 100, для ширины такие же
Позитивный:
    1,2,99,100
Негативный:
    0,101

Тест кейсы (6 позитивных и 3 негативных)
- 1 х 1
- 100 х 100
- 2 х 2
- 99 х 99
- 1 х 100
- 55 х 77

- 0 х 0
- 101 х 101
- (-1) х (-1)
